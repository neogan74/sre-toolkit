Kubernetes Health • Diagnostics • Audit

See the cluster the way an incident commander needs to.

k8s-doctor compresses the first stretch of Kubernetes triage into one CLI: health checks for posture, diagnostics for broken workloads, and audits for the policies that quietly create future outages.

Core Flows

Built for the first ten minutes of cluster trouble.

The value here is not “another kubectl wrapper.” The value is collapsing posture checks, failure analysis, and preventive audits into one predictable command surface.

Healthcheck

Read node state, pod distribution, and core component status when you need a clean baseline.

Posture

Diagnostics

Surface CrashLoopBackOff, restart storms, event pressure, and resource or probe gaps in one report.

Triage

Audit

Catch risky RBAC bindings, insecure security contexts, missing quotas, and weak namespace isolation.

Prevention
Command Pack

Three ways teams use it.

healthcheck 
k8s-doctor healthcheck --namespace platform --output table
diagnostics 
k8s-doctor diagnostics --namespace payments --output json
audit 
k8s-doctor audit --namespace payments --output table

Why operators keep it close

It fits into runbooks naturally: start broad, focus on one namespace, and export machine-readable data when the incident moves into automation.
