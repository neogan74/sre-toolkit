Load Testing • Resilience • Tail Latency

Push systems on purpose before production does it for you.

chaos-load is the toolkit’s practical stress utility: lightweight HTTP load generation, auth-aware endpoint checks, latency-focused reporting, and a clean base for future chaos scenarios like 5xx, jitter, explicit 504 timeouts, and dropped connections.

  --bearer-token $API_TOKEN \

Request caps Run bounded workloads or sustained duration-based stress tests.

Chaos path Inject latency, HTTP failures, timeout responses, and connection drops from the built-in mock server.
Fit

Use it when full load rigs are too much friction.

`chaos-load` is useful when you need a fast confidence check: can the endpoint take more traffic, where does latency bend, and how noisy do failures become under pressure?

Smoke load

Quick pre-release checks for throughput and error rate without setting up JMeter-sized machinery.

Fast feedback

Tail latency checks

Focus on p95 and p99 to see where user experience starts to degrade under pressure.

Performance reality

Protected APIs

Hit internal or staging routes guarded by Bearer tokens or Basic auth using first-class flags.

Auth aware

Chaos runway

Acts as the base layer for deliberate failure injection, including timeout and abrupt disconnect scenarios.

Evolution path
Command Pack

Three useful starting points.

smoke 
chaos-load http --url https://example.com --duration 30s --concurrency 10
stress 
chaos-load http --url https://api.example.com --duration 2m --concurrency 100
authenticated 
chaos-load http --url https://staging.example.com/internal/status --basic-username sre --basic-password changeme --requests 200
chaos mock 
chaos-load mock --port 8080 --error-rate 20 --latency 150ms --timeout-rate 5 --timeout-duration 12s --connection-failure-rate 10

Natural pairing

Run `chaos-load` to apply pressure, then use `k8s-doctor` to inspect how the cluster and workloads react.

Install

Build and run in minutes.

setup 
git clone https://github.com/neogan/sre-toolkit.git
cd sre-toolkit
make chaos-load

./bin/chaos-load http --url https://example.com --duration 30s --concurrency 10
