Prometheus • Alertmanager • Noise Reduction

Reduce alert fatigue with evidence, not vibes.

alert-analyzer turns raw alert history into something operationally useful: noisiest alerts, flapping candidates, correlated alert pairs, and machine-readable summaries for rule reviews.

What It Solves

From “too many alerts” to an actual shortlist.

If you already know you have alert fatigue, the next problem is prioritization. This tool exists to rank the offenders and give you data for threshold, `for:`, and signal-to-noise conversations.

Noisy alert ranking

Identify the alerts with the highest firing counts so review effort goes where it matters.

Prioritization

Flapping detection

Catch unstable rules that constantly switch state and erode operator trust.

Stability

Correlation analysis

Find alert pairs that repeatedly overlap so teams can investigate common causes instead of isolated symptoms.

Failure domains

Exportable summaries

Use JSON output to feed audits, dashboards, or change-review notes.

Automation
Command Pack

Three practical runs.

default analysis 
alert-analyzer analyze --prometheus-url http://prom:9090
flapping + correlation 
alert-analyzer analyze --prometheus-url http://prom:9090 --lookback 14d --show-flapping --show-correlation
automation 
alert-analyzer analyze --prometheus-url http://prom:9090 --output json --show-correlation

Where it fits

Use it in weekly alert review, postmortem follow-up, or as prep before changing rule thresholds.

Install

Run it against your alert history.

setup 
git clone https://github.com/neogan/sre-toolkit.git
cd sre-toolkit
make build-all

./bin/alert-analyzer analyze --prometheus-url http://localhost:9090
