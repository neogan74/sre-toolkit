Kubernetes CLI • Healthcheck • Diagnostics • Audit

Find the cluster problems before the pager does.

k8s-doctor is a production-minded command line utility for SRE teams that need a fast read on cluster health, noisy workloads, and policy gaps. It speaks fluent terminal, exports JSON for CI, and now ships with audit checks for probes, security context, RBAC, network policies, and resource quotas.

Why This Exists

One CLI for the first ten minutes of triage.

Most Kubernetes incidents waste time in tool switching: cluster health in one place, pod failures in another, security posture in a spreadsheet, and CI checks nowhere. k8s-doctor compresses that first response loop into one command surface.

Fast to run Easy to automate Opinionated enough to matter

Healthcheck

Readiness of nodes, pods, and control plane components without diving into multiple kubectl views.

For cluster posture

Diagnostics

CrashLoopBackOff, ImagePullBackOff, restart storms, events, and workload-level audit findings in one summary.

For incident triage

Audit

Probe coverage, security context gaps, risky RBAC grants, missing network segmentation, and absent quotas.

For prevention

CI Output

Table output for operators, JSON for machines, and non-zero exits when warnings or critical findings matter.

Feature Coverage

              
Built for SRE workflows, not screenshot demos.

              

                The pitch is not “observability, but prettier.” The pitch is fewer context switches,
                more predictable output, and a CLI that can become part of incident docs, platform audits,
                and CI guardrails.
              

            

          


          

            

              
Cluster Signals

              

                Node readiness, component health, pod phase distribution, and namespace-aware filtering
                for focused checks during partial outages.
              

            

            

              
Workload Signals

              

                Problem pod analysis with restart thresholds, event correlation, and resource/probe/security
                findings that surface why the workload is fragile.
              

            

            

              
Policy Signals

              

                Audit output now spans risky RBAC rules and bindings, missing network policies, and namespaces
                with no resource quotas.
              

            

          

        

      


      

        

          

            

              
How Teams Use It

              
Three concrete entry points.

            

          


          

            

              
During incidents

              
Run `diagnostics` when the cluster looks noisy but the failure mode is unclear.

            

            

              
Before releases

              
Run `audit --output json` in CI to reject workloads with predictable platform or security gaps.

            

            

              
For platform reviews

              
Use namespace-scoped audits to quickly compare team environments and spot policy drift.

Install

                
Fastest path to trying it.

                
Open the repo, build from source, or install just the CLI binary.

              

            

            

              

                
                setup
              

              
git clone https://github.com/neogan/sre-toolkit.git
cd sre-toolkit
make build

# or install directly
go install github.com/neogan/sre-toolkit/cmd/k8s-doctor@latest

# first run
k8s-doctor healthcheck --output table
k8s-doctor diagnostics --namespace payments
k8s-doctor audit --namespace payments --output json
